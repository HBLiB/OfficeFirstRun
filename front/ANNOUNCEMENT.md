# Round 12 — Loading Spinner Component

## Task

Create a small, reusable `LoadingSpinner` component and integrate it into existing pages that fetch async data (Services, Projects, Case Studies, Topology). 

## Requirements

1. Add `src/components/LoadingSpinner.jsx` — a simple CSS-based spinner (no extra dependencies).
2. Add matching styles (inline or a small CSS module).
3. Use `LoadingSpinner` in at least one page that currently fetches data, replacing any bare "Loading..." text.
4. Keep it minimal — no skeleton screens, no shimmer, just a clean spinner with optional label text prop.

## Scope

- Only touch files in `front/`.
- This is pure UI polish — no API changes.
